# GDScript 格式规则

来源：[Godot 4.x 官方风格指南](https://docs.godotengine.org/en/stable/tutorials/scripting/gdscript/gdscript_styleguide.html)。

## 编码与缩进

- 换行符 **LF**（非 CRLF/CR）；文件末尾 **一个** 换行符；编码 **UTF-8 无 BOM**
- 缩进使用 **制表符（Tab）**，每级缩进必须大于包含块

## 行尾逗号

多行数组/字典/枚举末尾**必须有逗号**，单行列表不加尾逗号。

```gdscript
# 正确
var array = [1, 2, 3]
var array = [
    1,
    2,
    3,
]
```

## 续行缩进

- 常规续行：**2 个缩进级别**
- 数组/字典/枚举续行：**1 个缩进级别**

```gdscript
# 常规续行 — 2 级
effect.interpolate_property(sprite, "transform/scale",
        sprite.get_scale(), Vector2(2.0, 2.0), 0.3,
        Tween.TRANS_QUAD, Tween.EASE_OUT)

# 数组续行 — 1 级
var party = [
    "Godot",
    "Godette",
    "Steve",
]
```

## 空白行与行长度

- 函数和类定义之间：**2 个空行**
- 函数内部逻辑片段之间：**1 个空行**
- 行长度硬限制 **100 字符**，软目标 **80 字符**

## 语句

- **一条语句一行**（三元运算符除外）
- 多行 if/三元拆分时缩进 **2 级**，优先用**括号**（非反斜杠）
- `and`/`or` 放下一行**开头**；避免不必要的圆括号

```gdscript
next_state = "idle" if is_on_floor() else "fall"

if (
        position.x > 200 and position.x < 400
        and position.y > 300 and position.y < 400
):
    pass
```

## 布尔运算

用英文关键字：`and` / `or` / `not`，**不**用 `&&` / `||` / `!`。

## 注释

- `#` / `##` 后**一个空格**；被注释掉的代码**无空格**（区分文本与停用代码）
- `#region` / `#endregion` 井号后**无空格**
- 优先独立行注释，内联注释仅用于短语

## 空格

- 运算符前后各一个空格，逗号后一个空格
- 字典引用/函数调用**无多余空格**
- 单行字典 `{ key = "value" }`（大括号内侧各一个空格）
- **不要**用空格垂直对齐表达式

## 引号与数字

- 默认**双引号**，单引号仅当减少转义时使用
- 浮点数保留前导/后缀零（`0.234`、`13.0`）
- 十六进制**小写**（`0xfb8c0b`）
- 大数字用下划线分隔（`1_234_567_890`），100 万以下不需要

## 审查清单

- [ ] LF 换行，文件末尾一个换行符，UTF-8 无 BOM
- [ ] Tab 缩进
- [ ] 多行数组/字典/枚举末尾逗号
- [ ] 续行缩进正确（常规 2 级，数组/字典/枚举 1 级）
- [ ] 函数/类间 2 空行，函数内逻辑片段 1 空行
- [ ] 行长度 ≤ 100 字符
- [ ] 一条语句一行（三元除外）
- [ ] 英文布尔关键字（and/or/not）
- [ ] 注释井号后有空格，被注释代码无空格
- [ ] 浮点数前导/后缀零，十六进制小写，大数字下划线分隔
