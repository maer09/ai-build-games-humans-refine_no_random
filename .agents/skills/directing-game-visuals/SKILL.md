使用此技能让游戏视觉传达游玩状态，而不仅仅是装饰屏幕。

默认产出物：
- 如果已知项目目录，将完整的视觉方案放入 `<PROJECT_DIR>/VISUAL_DESIGN.md`。
- 如果项目已有 README 或设计文档中包含视觉/音频章节，保持该章节为简短摘要，仅在视觉方向变更时更新。
- 当后续将生成像素艺术素材时，需包含素材交接章节，说明哪些元素保持程序化、哪些变为素材、调色板约束以及可读性验收标准。
- 如果未提供视觉标签，则从概念的材料、几何、运动或氛围中推导 2-3 个"概念衍生"标签，并在 `VISUAL_DESIGN.md` 中标注。

核心规则：
- 在添加细节之前，先确立一个主角、一个主要危险信号和一个主要奖励信号。
- 将调色板颜色分配给游戏玩法角色。
- 反馈必须在不依赖 UI 文字的情况下清晰可辨：运动、形状、冲击、时机、对比度和声音线索应承载含义。
- 保持画面中心可读；将嘈杂纹理和次要运动推至边缘，除非机制有特殊要求。
- 避免使用常见的模板化符号，除非它们被游戏的视觉逻辑所改造。

工作流程：
1. 提取游戏概念或视觉约束所暗示的情绪、材料、几何和运动特征。
2. 为整个游戏综合出一句视觉短语。
3. 将机制映射到视觉状态变化和反馈效果。
4. 选择 3-5 色调色板，并明确每个颜色的游戏玩法角色。
5. 验证在无 HUD 文字的单帧静态画面中，主角/主要危险/主要奖励是否一目了然。

阅读 `references/visual-design-guide.md` 获取详细的模式表格、设计检查清单以及反通用视觉附录模板。
