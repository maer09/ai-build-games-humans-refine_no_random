当游戏需要有意图的文本样式、字体采用，或为分发进行许可证合规的字体打包时，使用此技能。

默认产物：
- `<PROJECT_DIR>/TYPOGRAPHY_DECISION.md` — 排版决策、角色映射、回退/采用状态、许可证状态。
- `<PROJECT_DIR>/VISUAL_DESIGN.md` — 简短的视觉策略摘要（角色、尺寸层级、颜色、事件文本效果）；不要在此重复字体来源或许可证详情。
- `<PROJECT_DIR>/assets/fonts/` — 仅打包已采用的字体字重，不要批量放入候选字体。
- `<PROJECT_DIR>/licenses/fonts/` — 已采用字体的完整许可证文本。
- `<PROJECT_DIR>/THIRD_PARTY_LICENSES.md` — 已采用的外部字体来源和许可证摘要。

更新现有项目时，在选择文件路径之前，先检查其当前的主题/样式目录、排版文档、字体目录和许可证目录。如果指定的项目路径不存在，生成一个基于标准路径的计划，并说明跳过了项目盘点。

在回退字体实现期间，明确记录尚未采用任何外部字体，不要创建字体许可证文件或第三方字体条目。

在字体采用期间，仅打包选定的字重。如果字体来自临时/下载文件夹，复制前须验证确切的来源版本和许可证；如果来源无法确认，需从权威来源重新获取仅选定的字重。

核心规则：
- 通过引擎的主题/样式系统集中管理字体、颜色和尺寸；尽量减少逐元素覆盖。
- 使用明确的角色：Heading（标题）、Info（信息）、Numeric（数值）、Emphasis（强调）。
- HUD 文本保持信息展示用途；强调样式仅用于事件。
- 得分、计时器和计数器使用等宽或固定宽度的数值呈现。
- 通过描边、阴影或对比色带，确保文本在运动或嘈杂背景上保持可读。
- 仅打包已采用的字体，并包含其许可证文本。
- 对于程序化渲染的效果文本（引擎画布 / `_draw` / 即时模式绘制 API），需显式处理基线定位、水平居中和描边/阴影的合成偏移；在实际导出目标中验证渲染结果。

工作流程：
1. 早期实现阶段使用引擎的默认/回退字体。
2. 在主题/样式资源中定义角色 token。
3. 在最终采用阶段，将候选字体与视觉方向和可读性进行对比。
4. 选择各角色所需的最少字重，然后仅打包这些文件。
5. 更新许可证文档并检查导出目标（例如确保非资源许可证文本文件在 Web 导出时不会被丢弃）。
